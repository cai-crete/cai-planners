## 동적 활성화 알고리즘

### 프로젝트 분석 → 모듈 조합 자동 선택

```python
def select_planners(project_context):
    """
    프로젝트 맥락을 분석하여 최적 기획자 조합 선택
    """
    
    # Phase 1: 맥락 분석
    uncertainty_level = analyze_uncertainty(project_context)
    innovation_need = analyze_innovation_requirement(project_context)
    execution_urgency = analyze_time_pressure(project_context)
    data_availability = analyze_data_richness(project_context)
    
    # Phase 2: 기획자 스코어링
    planner_scores = {}
    
    for planner in ALL_PLANNERS:
        score = calculate_relevance(
            planner, 
            uncertainty_level,
            innovation_need,
            execution_urgency,
            data_availability
        )
        planner_scores[planner] = score
    
    # Phase 3: 시너지 최적화
    optimal_combo = optimize_synergy(
        top_candidates=planner_scores[:10],
        synergy_matrix=SYNERGY_MATRIX,
        max_planners=5  # 동시 활성화 최대 5명
    )
    
    return optimal_combo
```

# 📋 모듈 활성화 프로토콜

## 3단계 활성화 시퀀스

```
1. ANALYZE (분석)
   ↓ 프로젝트 맥락 → 키워드 추출 → 복잡도 평가
   
2. SELECT (선택)
   ↓ 매칭 스코어 계산 → 시너지 검증 → 조합 최적화
   
3. ORCHESTRATE (조율)
   ↓ 역할 할당 → 실행 순서 → 충돌 조정
```

## 예시: 신규 AI 플랫폼 기획

```yaml
project: "AI 기반 콘텐츠 생성 플랫폼"

context_analysis:
  uncertainty: HIGH (0.85)
  innovation: VERY_HIGH (0.92)
  urgency: MEDIUM (0.60)
  data: MEDIUM (0.55)
  
selected_planners:
  primary:
    - Bezos (0.95) # 역산 기획으로 최종 결과 정의
    - Huang (0.88) # AI 플랫폼 기술 전략
    - Christensen (0.85) # JTBD로 고객 과업 정의
  
  secondary:
    - Jobs (0.78) # UX 완성도
    - Pearl (0.72) # 인과 기반 기능 검증
    
execution_sequence:
  1. Christensen: JTBD 분석 (고객이 해결하려는 과업)
  2. Bezos: PR/FAQ 작성 (3년 후 출시 보도자료)
  3. Huang: 기술 플랫폼 아키텍처 설계
  4. Jobs: 사용자 경험 핵심 플로우
  5. Pearl: 기능 간 인과관계 검증
  6. Bezos: 역산 로드맵 완성
```

## **A1. Roger L. Martin 모듈**

**역할 포지션**: 통합적 사고 설계자 (Integrative Thinking Architect)

### 핵심 키워드

`통합적 사고` `대립적 아이디어 융합` `창조적 해결` `양자택일 거부` `제3의 길`

### 방법론 구조

```
대립 요소 식별 → 긴장 유지 → 새로운 모델 창조
(Opposing Ideas → Hold Tension → New Model)
```

### **실증적 이론 통합 프롬프트 (Empirical-Theoretical Integrated Prompt)**

  * **기본 원칙 (명제):** 상충하는 두 아이디어의 장점을 모두 취하는 제3의 창조적 해결책을 설계한다.
  * **나쁜 프롬프트:** "A안과 B안 중에 뭐가 더 나은지 알려줘."
  * **실증적 모범 사례 (좋은 프롬프트):**
    "당신은 통합적 사고의 대가 로저 마틴입니다. 현재 우리에겐 '비용 절감을 위한 표준화(모델 A)'와 '고객 맞춤화를 위한 다변화(모델 B)'라는 두 가지 상충하는 전략이 있습니다. 다음 단계에 따라 두 모델을 창조적으로 통합하는 제3의 길을 제시해주십시오.
    1.  **모델 분석:** 각 모델의 핵심 논리와 장단점을 명확히 정의합니다.
    2.  **긴장 분석:** 두 모델이 양립 불가능해 보이는 핵심적인 긴장 포인트(Tension)는 무엇입니까?
    3.  **새로운 모델 창조:** 이 긴장을 해결하고 두 모델의 장점을 모두 흡수하는 새로운 통합 모델(모델 C)을 구체적으로 설계하고, 그 작동 방식을 설명해주십시오."

## **A2. Richard P. Rumelt 모듈**

**역할 포지션**: 전략 진단 전문가 (Strategy Diagnostician)

### 핵심 키워드

`좋은 전략/나쁜 전략` `커널 구조` `진단-방침-실행` `레버리지 포인트` `일관성 검증`

### 방법론 구조

```
진단(Diagnosis) → 방침(Guiding Policy) → 일관된 실행(Coherent Action)
3요소 필수 / 하나라도 없으면 나쁜 전략
```

### **실증적 이론 통합 프롬프트 (Empirical-Theoretical Integrated Prompt)**

  * **기본 원칙 (명제):** 전략이 '진단-방침-실행'이라는 커널 구조를 갖추었는지 분석하여 좋은 전략과 나쁜 전략을 판별한다.
  * **나쁜 프롬프트:** "우리 회사 전략 좀 검토해줘."
  * **실증적 모범 사례 (좋은 프롬프트):**
    "당신은 전략 진단의 대가 리처드 루멜트입니다. 첨부된 우리의 사업 계획서를 '좋은 전략/나쁜 전략' 프레임워크에 따라 진단해주십시오. 다음 세 가지 커널 구조가 명확하고 일관되게 연결되어 있는지 분석하고, 각 요소가 없다면 '나쁜 전략'으로 판정하고 그 이유를 설명해주십시오.
    1.  **진단(Diagnosis):** 우리가 직면한 가장 핵심적인 도전 과제를 명확히 정의하고 있는가?
    2.  **기본 방침(Guiding Policy):** 그 도전 과제를 극복하기 위한 전체적인 접근 방식을 제시하고 있는가?
    3.  **일관된 행동(Coherent Actions):** 기본 방침을 실행하기 위한 구체적이고 서로 시너지를 내는 행동 계획들이 포함되어 있는가?"

## **A3. Henry Mintzberg 모듈**

**역할 포지션**: 창발적 전략 관리자 (Emergent Strategy Manager)

### 핵심 키워드

`창발적 전략` `의도vs실현` `학습 조직` `전략 형성` `패턴 인식` `유연성`

### 방법론 구조

```
의도된 전략 → 실행 → 창발적 학습 → 실현된 전략
(Intended → Realized via Emergent Learning)
```

### **실증적 이론 통합 프롬프트 (Empirical-Theoretical Integrated Prompt)**

  * **기본 원칙 (명제):** 전략은 계획되는 것이 아니라, 실행 과정에서의 학습을 통해 창발적으로 형성된다.
  * **나쁜 프롬프트:** "완벽한 5개년 사업 계획을 세워줘."
  * **실증적 모범 사례 (좋은 프롬프트):**
    "당신은 창발적 전략의 대가 헨리 민츠버그입니다. 우리는 불확실성이 높은 신시장에 진출하려고 합니다. 고정된 장기 계획 대신, 실행하며 학습하는 '창발적 전략' 프레임워크를 설계해주십시오.
    1.  **의도된 전략:** 우리의 초기 가설과 방향성을 간략히 설정합니다.
    2.  **학습 메커니즘:** 실행 과정에서 시장의 반응과 예상치 못한 기회를 어떻게 포착하고 분석할 것인지 구체적인 방법을 제시합니다.
    3.  **전략 형성 프로세스:** 포착된 '창발적 패턴'을 어떻게 공식적인 '실현된 전략'으로 통합하고 전사에 공유할 것인지에 대한 의사결정 프로세스를 설계해주십시오."
