# PROTOCOL: SQUAD KEYWORD EXTRACTION (V3.1 - Quantitative Calibration)

본 프로토콜은 기획자별 키워드 추출의 '밀도'와 '정확도'를 정량적으로 제어하여, 시스템의 가독성을 매트릭스 차원에서 보정하기 위한 기술 지침이다.

## 1. 키워드 추출 규격 (Extraction Specifications)

모든 전문가는 아래의 정량적 제약 사항을 100% 준수하여 키워드를 산출한다.

- **Length (길이):** 제한 없음. (단일 단어부터 의미적 구절까지 유연하게 허용)
- **Density (밀도):** 각 추출 대상 내 '실질 형태소(명사, 수사, 용언의 어근)'의 비중이 80% 이상이어야 함. (조사 및 어미 등 문법적 군더더기 배제)
- **Position (위치):** 발언 텍스트의 [초입부 20%] 또는 [종결부 20%] 공간에서 발견되는 어구에 우선권 부여.
- **Quantity (수량):** 발언당 최소 3개, 최대 4개의 핵심 포인트로 제한.

## 2. 모드별 정량적 정밀 필터 (Mode-Specific Quantitative Filters)

### 🌈 Mode A (Creative) - [Entropy Filter]
- **기준:** '고유 컨셉명(Named Concept)' 및 '신조어적 결합'이 포함된 대상 비중 60% 이상.
- **목표:** 창발적 명칭 강조.

### 🧠 Mode B (Logical) - [Structural Filter]
- **기준:** '메커니즘', '계열화', '인과' 등 구조적 연결 명사가 포함된 대상 2개 이상 필수.
- **목표:** 시스템적 구조 강조.

### 🛠️ Mode C (Practical) - [Accuracy Filter]
- **기준:** '수치(Number)', '단위(Unit)', '기술 자산(Tech Assets)'이 포함된 대상 1개 이상 필수 포함.
- **목표:** 실행 데이터 포인트 강조.

## 3. 역할 x 모드 통합 매트릭스 (Matrix Mapping)

| 역할 \ 지표 | **Mode A (Novelty)** | **Mode B (Logic)** | **Mode C (Spec)** |
| :--- | :---: | :---: | :---: |
| **제안자** | 컨셉 명칭 (1.0) | 프레임워크 (1.0) | 실행 엔진 (1.0) |
| **반박자** | 기회비용 (1.2) | 논리 모순 (1.2) | 기술/비용 제약 (1.2) |
| **통합자** | 시너지 명제 (1.5) | 상위 로직 (1.5) | 자원 배분안 (1.5) |
| **검증자** | 가치 지속 지표 (2.0) | 인과 정합도 (2.0) | KPI/수치 사양 (2.0) |

## 4. 품질 검사 (QA Checklist)
- [ ] 실질 형태소 밀도가 80%를 상회하는가? (불필요한 조사가 포함되지 않았는가?)
- [ ] 모드별 필수 포함 요소(신조어/구조어/수치)가 반영되었는가?
- [ ] 발언당 3~4개의 핵심 포인트로 압축되었는가?
