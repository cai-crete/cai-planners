# PRD: IMAGE TO ELEVATION (ES-MoE 기반 다중 전문가 토론 툴)

## 1. 🎯 프로젝트 개요 (Overview)
* **목적:** 사용자가 입력한 텍스트 및 이미지(건축 스케치, 실사 등)를 기반으로, 시스템에 내장된 14명의 가상 전문가(ES-MoE 아키텍처)가 [제안 → 반박 → 통합]의 변증법적 토론을 거쳐 최적의 결과값(기획안, 디자인 방향성 등)을 도출하는 프론트엔드 기반 웹 애플리케이션입니다.
* **핵심 가치:** 단방향 AI 답변이 아닌, 다수 전문가의 격론을 시각적인 노드(Node) 트리 형태로 제공하여 기획의 논리적 사각지대를 제거하고 사용자에게 완벽한 통제권과 확장성을 부여합니다. [cite_start]주니어 개발자도 명확히 이해하고 구현할 수 있도록 직관적이고 구체적으로 작성되었습니다. [cite: 7]
* **타겟 환경:** PC, 태블릿, 모바일 웹 브라우저 (반응형 지원).

## 2. 🎨 UI/UX 및 디자인 정책 (Design Policy)
* **톤앤매너:** 미니멀리즘 기반의 블랙 & 화이트 (Black/White).
* **애니메이션:** 차분하고 부드러운 인터랙션 지향.
    * **노드 생성 시:** 서서히 나타나는 페이드인(Fade-in) 시퀀스 효과.
    * **로딩 상태:** "OOO 전문가가 분석 중입니다" 텍스트와 함께 점 세 개(...)가 튀어 오르거나 맥박 형태의 선율 애니메이션 적용.
* **노드 시각화 (상태별 구분):** '제안(Thesis)', '반박(Antithesis)', '통합(Synthesis)' 노드가 한눈에 구분되도록 형태나 선 굵기 등의 직관적 디자인 요소 적용.

## 3. ⚙️ 핵심 기능 요구사항 (Core Features)

### 3.1 무한 캔버스 (Infinite Canvas Workspace)
* **화면 조작 (Navigation):** 웹 기본 우클릭 메뉴(Context Menu)를 비활성화합니다.
    * **이동 (Pan):** 마우스 우클릭 드래그 / 터치패드 2손가락 탭 및 드래그.
    * **확대/축소 (Zoom):** 마우스 휠 / 2손가락 핀치 줌.
    * **선택 (Select):** 좌클릭 / 1번 터치.
* **유틸리티:** 화면 우측 하단 줌 컨트롤 옆에 '캔버스 중앙으로 이동(Recenter)' 버튼 배치.

### 3.2 스티키 메모 및 자산 가져오기 (Import & Sticky Notes)
* 좌측 탭을 통해 캔버스에 텍스트와 이미지를 '스티키 메모' 형태로 자유롭게 배치할 수 있습니다.
* **Enhanced Prompt (프롬프트 강화):** 텍스트 스티키 메모 내부에 ✨(반짝임 아이콘) 버튼을 제공합니다. 클릭 시 입력된 텍스트 길이에 따라 AI가 자동으로 내용을 요약하거나 토론에 적합하게 살을 붙여줍니다.
* **수정 및 잠금 (Lock):** 스티키 메모는 언제든 더블클릭하여 수정 가능하나, GENERATE(생성)가 진행되는 동안에는 내용이 잠금(Lock) 처리됩니다.

### 3.3 요소 그룹핑 및 토론 트리거 (Grouping & GENERATE)
스티키 메모들을 엮어 토론을 시작하는 방식은 다음 두 가지를 모두 지원합니다.
1. **다중 선택 및 플로팅 바:** 여러 스티키 메모를 마우스 드래그 및 Shift+클릭으로 다중 선택하면 나타나는 플로팅 바에서 `[✨ GENERATE]` 버튼 클릭.
2. **연결선 (Link):** 스티키 메모 간에 선을 직접 그려 연결한 후, 생성된 연결 그룹 영역의 `[✨ GENERATE]` 버튼 클릭.

### 3.4 ES-MoE 기반 토론 엔진 (Discussion Engine)
* **턴(Turn) 진행:** GENERATE 실행 시 [제안 → 반박 → 통합] 순으로 노드가 순차적으로 생성되며 선으로 연결됩니다. 최대 허용 턴 수는 1회 당 '3턴'으로 제한합니다.
* **분화 (Branching):** 사용자가 캔버스 상의 특정 과거 노드를 클릭하고 다시 GENERATE를 누르면, 뿌리부터 해당 노드까지의 '단일 경로 대화'만을 맥락으로 삼아 새로운 토론 가지(Branch)가 생성됩니다. (과거 파생된 노드들은 보존됨)
* **오류 및 중단 제어:**
    * **정지 (Stop):** 생성 도중 사용자가 정지 버튼을 누르면 마지막으로 완료된 노드까지만 캔버스에 저장되고 프로세스가 즉시 중단됩니다.
    * **재시도 (Retry):** API 실패 시 화면 하단에 알림(Toast)이 뜨고, 캔버스 상의 실패한 빈 노드에 눈에 띄는 '재시도(Retry)' 버튼이 활성화됩니다.

### 3.5 좌측 컨트롤 탭 (Left Tab - Control Center)
캔버스 좌측에서 불투명(Solid)하게 화면을 덮어쓰며 열리는 패널입니다.
* **[상단] CANVAS LIBRARY:** '새 캔버스(New Project)' 생성 버튼 및 IndexedDB에 저장된 과거 프로젝트(대화 캔버스) 목록 불러오기.
* **[중앙] 전문가 설정:**
    * 자동 차출 / 수동 선택 모드 토글.
    * 수동 선택 시 14명의 전문가 리스트(텍스트 및 아이콘) 노출 및 체크박스 선택 가능. (※ 추후 확장을 위해 데이터 구조상 `avatarUrl` 필드를 미리 할당해 둡니다.)
* **[하단] 상태창:** 현재 API 턴 수(예: 1/3 진행 중) 및 Enhanced Prompt 설정 표기.

### 3.6 우측 결과 확인 탭 (Right Tab - Detail View)
노드 클릭 시 우측에서 불투명(Solid)하게 덮어쓰며 열리는 패널입니다.
* **대화 내용 렌더링:** 전문가의 초상화/아이콘, 이름과 함께 해당 턴의 상세 대화 내용이 마크다운(Markdown) 형태로 렌더링됩니다.
* **기능 버튼:** * `[복사 (Copy)]`: 텍스트 클립보드 복사.
    * `[내보내기 (Export)]`: 선택한 최종 결론 노드까지 이어지는 **'단일 직선 경로(뿌리부터 해당 노드까지의 스토리)'** 대화 내용만 PDF, MD, TXT 형태로 다운로드.

## 4. 🛠️ 기술 스택 및 데이터 정책 (Tech Stack & Data)
* **프론트엔드 환경:** Vite + React. 상태 관리는 Zustand, 스타일링은 Tailwind CSS를 사용합니다. 무한 캔버스는 React Flow 라이브러리를 활용합니다.
* **로컬 저장소 (Local Storage):** 브라우저 내장 데이터베이스인 **IndexedDB**를 사용하여 대용량의 프로젝트 데이터와 캔버스 노드 정보를 서버 없이 무조건 로컬에 영구 저장합니다.
* **API 통신 (Gemini):**
    * 컨텍스트 최적화: 새로운 노드 생성 시 비용 절감을 위해 직전 3~4개의 핵심 노드 컨텍스트만 선별하여 전송합니다.
    * 이미지 데이터: 브라우저 단에서 Base64 문자열로 인코딩하여 프롬프트와 함께 API로 전송합니다.
* **보안 및 프롬프트 관리:** 14명 기획자의 페르소나 및 핵심 프로토콜 정보는 소스 코드의 환경 변수(ENV)에 고정(Hardcoding)하여 배포합니다.
