# Re-generate Protocol — 기획자들 V3.1 (재토론 규약)

## 1. 프로토콜 목적
본 규약은 사용자가 기존 토론 그룹 노드(Turn Group Node)의 [PROMPT] 창에 새로운 지시사항을 입력하고 Re-generate 버튼을 누를 때 발동되는 재토론 규약이다.
단순한 재생성이 아닌, 이전 기획서의 결론을 출발점으로 삼아 특정 파트를 심화(Deep Dive)하거나 방향을 수정(Pivot)하는 구조적 2차 토론을 목표로 한다.

## 2. 입력값 정의
Re-generate 호출 시 다음 두 가지 정보가 반드시 엔진에 투입된다:

- **[이전 최종 기획서 (prevFinalOutput)]**: 직전 Turn Group Node의 finalOutput 마크다운 전체.
- **[사용자 새 지시사항 (newPrompt)]**: 사용자가 PROMPT 창에 직접 입력한 자연어 피드백 또는 새로운 방향성.

## 3. 컨텍스트 구성 원칙
엔진은 두 입력값을 다음 순서로 조합하여 새로운 토론 컨텍스트를 구성한다:

1. [이전 토론의 최종 기획서] 섹션에 prevFinalOutput 원문을 그대로 삽입한다.
2. [사용자의 새로운 지시사항 / 피드백] 섹션에 newPrompt를 삽입한다.
3. 엔진은 반드시 이전 기획서를 '상속'하고, 새 지시사항을 '수정 명령'으로 받아들여야 한다.
   → 이전 결론을 완전히 무시하거나 처음부터 다시 시작하는 것은 엄격히 금지된다.

## 4. 재토론 4인 스쿼드 역할 정의
기존 4인 스쿼드(Thesis → Antithesis → Support → Synthesis)의 역할은 유지되되, 다음의 추가 규칙이 적용된다:

- **Thesis (제안)**: 이전 기획서에서 새 지시사항과 가장 부합하는 방향성을 선정하여 기반으로 삼고, 그 방향으로 심화된 새 제안을 전개한다.
- **Antithesis (반박)**: 새 지시사항대로 변경했을 때 발생할 수 있는 현실적 제약(예산, 일정, 실행력)을 비판적으로 검토한다.
- **Support (검증)**: 이전 기획서의 결론과 새 지시사항 사이의 인과적 충돌 또는 시너지를 규명한다.
- **Synthesis (통합)**: 이전 기획서를 기반으로 새 지시사항을 반영하여 수정된 최종 기획서를 도출한다.

## 5. 출력 형식 (Output Format)
기본 생성(`generateDiscussion`) 함수와 동일한 JSON 스키마(DiscussionResult)를 사용한다.
모든 shortContent와 fullContent는 자연어 대화체로, finalOutput만 순수 마크다운 기획서 형식으로 출력한다.

## 6. 금지 사항
- 이전 기획서의 핵심 결론을 근거 없이 폐기하는 것은 금지.
- HTML 태그 사용 절대 금지.
- shortContent, fullContent에 마크다운 기호 사용 금지.
- 이전 기획서에 없던 완전히 새로운 주제로 이탈(Topic Drift)하는 것은 금지.
